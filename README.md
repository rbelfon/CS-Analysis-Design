# CS-Analysis-Design
Course planner program using a binary search tree to load, sort, and search course data from a file.
## Project Overview

In this course, I worked on building a program to help academic advisors quickly access course information. The goal was to read course data from a file, organize it properly, and allow users to print and search for courses easily.

## Problem I Was Solving

The main problem was figuring out how to store and manage course data in a way that is efficient and easy to use. The program needed to display courses in order and allow users to search for specific courses, so choosing the right data structure was important.

## My Approach

I broke the problem down step by step. First, I focused on reading and understanding the file input. Then I worked on how to store the data using different data structures like a vector, hash table, and binary search tree.

I chose the binary search tree because it keeps the data sorted automatically, which makes printing the course list easier and more efficient without needing extra sorting.

## Challenges I Faced

One of the biggest challenges was getting the file input to work correctly and making sure the data was actually stored and available after loading. I also ran into issues with how the program handled user input and menu options.

I worked through these problems by testing each part step by step and fixing one issue at a time instead of trying to solve everything at once.

## What I Learned

This project helped me understand how important data structures are when designing a program. The structure you choose directly affects how efficient your program is.

It also changed how I approach coding. I now think more about planning before writing code and making sure everything is organized in a way that is easy to follow.

## Final Thoughts

Overall, this project improved my problem-solving skills and helped me build a better understanding of how to design programs that are efficient, readable, and maintainable.
